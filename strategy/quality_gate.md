## 🎯 系统目标

**核心原则**: 不合格就关门，不感情用事

- ✅ 数据驱动 - 客观指标判断，不凭感觉
- ✅ 自动风控 - 亏损品种自动停止交易
- ✅ 策略筛选 - 只用达标策略
- ✅ 避免过度交易 - 抑制不合格策略

---

## 📊 策略健康状态

| 状态 | 标识 | 含义 | 动作 |
|------|------|------|------|
| **ACTIVE** | 🟢 | 策略健康，绩效达标 | 正常交易 |
| **OPTIMIZING** | 🟡 | 需优化，绩效一般 | 暂停，回测调参 |
| **STOPPED** | 🔴 | 不合格，连续亏损 | 彻底停止 |
| **PENDING** | ⏳ | 新策略，测试中 | 轻仓测试 |

## 📋 质量指标门禁

### 核心指标 (必须全部达标)

| 指标 | 达标标准 (🟢) | 警告线 (🟡) | 停止线 (🔴) | 评估周期 |
|------|---------------|-------------|-------------|----------|
| **胜率** | ≥45% | 35-45% | <35% | 20 单后 |
| **盈利因子** | ≥1.5 | 1.0-1.5 | <1.0 | 20 单后 |
| **最大回撤** | ≤10% | 10-15% | >15% | 实时 |
| **夏普比率** | ≥1.0 | 0.5-1.0 | <0.5 | 月度 |
| **交易数量** | ≥20 单/月 | 10-20 单 | <10 单 | 月度 |
| **平均盈亏比** | ≥1:1.5 | 1:1-1:1.5 | <1:1 | 10 单后 |

### 辅助指标 (参考)

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 最大连续亏损 | <5 单 | 连续亏损不超过 5 单 |
| 最大连续盈利 | >3 单 | 鼓励连胜 |
| 平均持仓时间 | 2-24 小时 | 避免过长/过短 |
| 日交易频率 | 2-5 单 | 避免过度交易 |

## 🚨 自动风控规则

### 触发条件 → 自动执行

```
【规则 1】连续亏损 5 单
IF 连续亏损 ≥ 5 单
THEN 状态 → OPTIMIZING
     动作 → 暂停交易，回测分析
     通知 → 飞书汇报

【规则 2】单日亏损达 3%
IF 单日亏损 ≥ 3%
THEN 暂停交易 1 天
     通知 → 飞书汇报
     恢复 → 次日自动恢复

【规则 3】单周亏损达 5%
IF 单周亏损 ≥ 5%
THEN 状态 → OPTIMIZING
     动作 → 暂停交易，全面复盘
     通知 → 飞书汇报

【规则 4】单月亏损达 10%
IF 单月亏损 ≥ 10%
THEN 状态 → STOPPED
     动作 → 彻底停止，重新设计策略
     通知 → 飞书汇报 + 人工确认

【规则 5】胜率过低 (20 单后)
IF 交易数量 ≥ 20 AND 胜率 < 35%
THEN 状态 → STOPPED
     动作 → 停止策略，分析原因
     通知 → 飞书汇报 + 人工确认

【规则 6】盈利因子过低 (20 单后)
IF 交易数量 ≥ 20 AND 盈利因子 < 1.0
THEN 状态 → STOPPED
     动作 → 停止策略，亏多赚少不可接受
     通知 → 飞书汇报 + 人工确认

【规则 7】信号质量下降
IF 连续 3 日无达标信号
THEN 检查 → 扫描算法是否异常
     调整 → 临时降低门槛 20%
     通知 → 飞书汇报

【规则 8】保证金水平过低
IF 保证金水平 < 200%
THEN 暂停开新仓
     动作 → 等待保证金恢复
     通知 → 飞书预警
```

## 🔧 执行脚本

### 自动检查脚本 (check_quality_gate.py)

```python
#!/usr/bin/env python3
# 策略质量门禁自动检查

def check_quality_gate():
    """检查策略状态，自动执行风控"""
    
    # 获取交易数据
    stats = get_trading_stats()
    
    # 检查风控规则
    if stats['consecutive_losses'] >= 5:
        set_strategy_status('OPTIMIZING')
        send_alert('连续亏损 5 单，暂停交易')
        return
    
    if stats['daily_loss'] >= 0.03:
        pause_trading(24)  # 暂停 24 小时
        send_alert('单日亏损达 3%，暂停交易 1 天')
        return
    
    if stats['win_rate'] < 0.35 and stats['trade_count'] >= 20:
        set_strategy_status('STOPPED')
        send_alert('胜率低于 35%，停止策略')
        return
    
    if stats['profit_factor'] < 1.0 and stats['trade_count'] >= 20:
        set_strategy_status('STOPPED')
        send_alert('盈利因子低于 1.0，停止策略')
        return
    
    # 状态正常，继续交易
    if is_status_pending() and stats['trade_count'] >= 20:
        evaluate_pending_strategy(stats)
```
