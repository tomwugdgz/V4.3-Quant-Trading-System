### 4. 止损止盈设置分析 (当前 50/75 pips)

#### 当前设置问题
```
止损：50 pips (固定)
止盈：75 pips (固定)
盈亏比：1:1.5

问题:
1. 未考虑品种波动率差异
2. 未考虑时间段波动率变化
3. 未考虑当前市场状态 (高/低波动)
4. 固定盈亏比可能非最优
```

#### 基于 ATR 的动态止损建议

**ATR (Average True Range)** 是衡量波动率的最佳指标，建议:

```
止损 = ATR(14) × 1.5 至 2.0
止盈 = ATR(14) × 3.0 至 4.0
盈亏比 = 1:2 至 1:2.5
```

#### 各品种优化建议 (基于 1H 图 ATR)

| 品种 | 当前 ATR(14) | 建议止损 | 建议止盈 | 盈亏比 |
|------|--------------|----------|----------|--------|
| EURUSD | ~8 pips | **12-16 pips** (ATR×1.5-2) | **32-40 pips** (ATR×4) | 1:2.5 |
| GBPUSD | ~12 pips | **18-24 pips** | **48-60 pips** | 1:2.5 |
| USDJPY | ~10 pips | **15-20 pips** | **40-50 pips** | 1:2.5 |
| AUDUSD | ~9 pips | **14-18 pips** | **36-45 pips** | 1:2.5 |
| NZDUSD | ~10 pips | **15-20 pips** | **40-50 pips** | 1:2.5 |
| USDCHF | ~8 pips | **12-16 pips** | **32-40 pips** | 1:2.5 |

**对比当前**:
- EURUSD: 50→16 pips (降低 68%)，75→40 pips (降低 47%)
- **优势**: 止损更紧，单笔风险降低，可承受更多连续亏损
- **风险**: 可能被正常波动止损，需要更高胜率补偿

#### 分时段止损调整

### 5. 仓位大小计算优化

#### 当前方法 (固定风险金额)
```python
risk_amount = $50 (固定)
position_size = risk_amount / (|entry - sl| × contract_size)
```

**问题**:
- ❌ 所有品种统一 $50，未考虑波动率差异
- ❌ 高波动品种 (GBPJPY) 与低波动品种 (EURUSD) 风险相同
- ❌ 未考虑账户增长后的仓位调整

#### 优化方案 1: 基于波动率的动态风险

```python
# 基础风险金额
base_risk = account_balance × 0.5%  # 账户的 0.5%

# 波动率调整系数 (基于 ATR)
volatility_factor = ATR_current / ATR_average

# 调整后风险
adjusted_risk = base_risk / volatility_factor

# 高波动时降低风险，低波动时增加风险
```

#### 优化方案 2: 分级风险模型

| 品种分类 | 品种示例 | 风险比例 | 理由 |
|----------|----------|----------|------|
| **低风险** | EURUSD, USDJPY | 0.6% | 流动性高，趋势稳定 |
| **中风险** | GBPUSD, AUDUSD | 0.5% | 流动性中等 |
| **高风险** | GBPJPY, NZDUSD | 0.4% | 波动大，假突破多 |
| **禁止** | 加密货币 | 0% | 已禁用 |

#### 优化方案 3: 账户增长阶梯调整

| 账户余额 | 单笔风险% | 风险金额 | 理由 |
|----------|-----------|----------|------|
| <$10k | 0.5% | $50 | 本金小，保守为主 |
| $10k-$20k | 0.6% | $60-120 | 开始积累，适度增加 |
| $20k-$50k | 0.7% | $140-350 | 稳定盈利，增加敞口 |
| >$50k | 0.5% | $250+ | 本金大，降低比例保护 |
