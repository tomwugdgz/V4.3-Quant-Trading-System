## 前言：一周亏 $60 换来的教训

我的量化交易系统 V4.3 基于 MT5 + Python，使用 ICMarkets 模拟账户，杠杆 1:5，本金 $10,000。

04/20-04/24 这一周，累计亏损 -$60.48（-0.60%）。金额不大，但问题严重——5 笔止损、2 笔止盈、胜率仅 25%，实际杠杆一度飙到 63 倍（设计值是 5 倍）。

问题不在市场，在系统。这篇文章记录了我如何定位 6 个 bug、重写信号算法、构建独立风控层，最终将系统从 V2 升级到 V3.0 的完整过程。

**重复持仓**: 同一品种同时持有 2 单（NZDUSD），定时任务并发导致，无文件锁保护。

## 二、Bug 修复清单

### P0 — 致命（影响每笔交易）

**Bug 1: 仓位计算 pip value 硬编码**

问题代码:
```python
# 错误：硬编码 pip value
pip_value_per_lot = 0.10  # 实际应为 ~10.00
```

修复后:
```python
# 正确：动态获取 MT5 品种信息
pip_size = symbol_info.point * 10 if symbol_info.digits in (3, 5) else symbol_info.point
pip_value_per_lot = symbol_info.trade_tick_value * (pip_size / symbol_info.trade_tick_size)
```

实际值验证:
- EURUSD: pip_size=0.0001, pip_value=$10.00/lot ✓
- USDJPY: pip_size=0.01, pip_value=$6.28/lot ✓
- NZDUSD: pip_size=0.0001, pip_value=$10.00/lot ✓

**Bug 2: JPY 品种止损硬编码**

问题:
```python
# 错误：所有品种统一除以 10000
sl_pips = atr * 10000  # JPY 品种应该是 100
```

修复:
```python
# 按 digits 自动判断
def get_pip_divisor(symbol_info):
    if symbol_info.digits == 3:  # JPY
        return 100
    return 10000
```

### P1 — 高优先级

**Bug 3: 信号算法分裂**

两套脚本使用完全不同的算法，信号强度差百倍。解决方案：统一为一套 V3 算法，两个脚本都调用同一个 `calculate_signal_v3()` 函数。

**Bug 4: 重复持仓（并发）**

定时任务每 15/20/30 分钟执行一次，上次没跑完下次又启动了。添加文件锁：

```python
import msvcrt  # Windows
# Linux 用 import fcntl

def acquire_lock():
    lock_fd = open('.monitor.lock', 'w')
    msvcrt.locking(lock_fd.fileno(), msvcrt.LK_NBLCK, 1)
    return lock_fd
```

### P2 — 中优先级

**Bug 5: 方向冲突** — 信号 SELL 但持仓 BUY，通过风控层校验解决。
**Bug 6: 自动开仓手数异常** — 仓位计算修复后自然解决。

## 三、策略升级：V2 → V3

### 3.1 V2 的问题

V2 策略过于简单，只看两个指标：
- SMA 交叉（5 日/20 日）
- RSI（超买超卖）

问题：
1. 单时间框架（只看 H1），无法识别大趋势
2. 无趋势强度过滤，震荡市频繁交易
3. 止损固定 30 pips，不随波动率调整
4. 无品种相关性过滤
5. 风控与信号混在一起

### 3.2 V3 多时间框架策略

**核心理念**: 大周期定方向，小周期找入场。

```
D1 (日线) → 确定交易方向 (BUY/SELL)
H4 (4小时) → 确认信号强度 (RSI + MACD)
H1 (1小时) → 精确入场点位 (ADX + ATR + 布林带)
```

**具体逻辑**:

1. **D1 趋势过滤器**
   - EMA20 > EMA50 且价格 > EMA20 → 多头
   - EMA20 < EMA50 且价格 < EMA20 → 空头
   - 否则 → 不交易

2. **H4 信号确认**
   - RSI < 35 且 D1 多头 → 超卖反弹 (+2 分)
   - RSI > 65 且 D1 空头 → 超买回落 (+2 分)
   - MACD 方向与 D1 一致 → (+1 分)

3. **H1 精确入场**
   - ADX > 25 → 趋势足够强 (+2 分)
   - 布林带突破 → (+1 分)
   - ADX < 25 → 震荡市，**不开仓**

4. **综合评分**
   - 总分 >= 5 → 强信号，开仓
   - 总分 >= 3 → 达标信号，可开仓
   - 总分 < 3 → 不交易

### 3.3 ATR 动态止损

不再用固定 30 pips 止损，改用 ATR（平均真实波动范围）：

```python
atr = calculate_atr(h1_data, period=14)
sl_pips = max(atr * 1.5, 15)  # 至少 15 pips
tp_pips = sl_pips * 2.0        # 盈亏比 1:2
```

效果:
- 波动大时止损宽，不容易被扫
- 波动小时止损窄，风险可控

### 3.4 ADX 趋势强度

ADX（平均趋向指数）衡量趋势强度，不衡量方向：
- ADX < 20 → 无趋势，震荡市
- ADX 20-25 → 趋势开始形成
- ADX > 25 → 趋势明确，**可以交易**
- ADX > 50 → 趋势非常强

V3 强制要求 ADX > 25 才开仓，过滤掉震荡市的无效信号。

### 3.5 品种相关性过滤

NZDUSD 和 AUDUSD 相关性 > 0.8，同时持有多单等于加倍下注。V3 加入相关性矩阵：

```python
CORRELATION_MAP = {
    "NZDUSD": ["AUDUSD"],
    "AUDUSD": ["NZDUSD"],
    "EURUSD": ["GBPUSD"],
    "GBPUSD": ["EURUSD"],
    "USDCHF": ["EURUSD", "GBPUSD"],
}
```

开仓前检查：如果已持有相关品种，拒绝开新仓。

### 3.6 独立风控层

V2 的风控和信号逻辑混在一起，V3 将风控独立为 8 项校验，**全部通过才执行**：

| # | 校验项 | 标准 |
|---|--------|------|
| 1 | 总持仓数 | < 3 单 |
| 2 | 品种重复 | 同一品种 < 1 单 |
| 3 | 日亏损 | 净值 > 余额的 97% |
| 4 | 实际杠杆 | < 3 倍 |
| 5 | 保证金水平 | > 200% |
| 6 | 品种相关性 | 与持仓相关性 < 0.7 |
| 7 | 冷却期 | 距上次平仓 > 30 分钟 |
| 8 | 信号强度 | >= 0.15% |

## 六、教训总结

1. **量级校验是必须的**: 涉及资金/仓位的计算，写完必须打印中间值验证。我在这个 bug 上亏了 10 天。

2. **信号算法只能有一套**: 多套并行维护一定会分裂，删掉冗余的。

3. **风控必须独立于信号**: 信号负责发现机会，风控负责否决交易。职责分离。

4. **先修架构，再优化算法**: 在漏水的船上加水是没用的。6 个 bug 里有 4 个是架构级问题。

5. **讨论 20 轮不如直接测试**: 对于影响明确的 P0 bug，直接修。架构讨论留给复杂问题。

6. **周末不开加密货币**: 流动性差，点差大，信号不可靠。周一再动手。
