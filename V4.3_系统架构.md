## 一、系统概览

### 1.1 架构目标

从 v2.0 的"高稳定性 + 低收益"升级为"稳定 + 正期望值"系统。

**核心改进**:
1. ✅ 市场状态判断 → 解决"策略结构偏随机"
2. ✅ 多因子评分 → 解决"无稳定 Edge"
3. ✅ 动态风控 → 解决"风控过强压制盈利"
4. ✅ 自动复盘 → 解决"复盘靠人工"

## 二、核心模块

### 2.1 Market Regime Agent

**职责**: 判断市场状态，动态调整阈值

**输入**:
- MT5 K 线数据（H1、D1）

**处理**:
1. 计算 ADX、EMA、ATR、布林带
2. 判断市场状态（4 种）
3. 输出动态阈值和仓位系数

**输出**:
```json
{
  "regime": "TRENDING_UP",
  "threshold": 0.08,
  "position_multiplier": 1.0,
  "should_trade": true
}
```

**文件**: `v4_3/market_regime.py`

---

### 2.2 Factor Score Engine

**职责**: 多因子评分，生成交易信号

**输入**:
- MT5 K 线数据
- 市场状态（来自 Regime）

**处理**:
1. 计算 4 因子（动量、均值回归、突破、波动率）
2. 加权综合评分（0-100）
3. 映射交易信号（5 级）

**输出**:
```json
{
  "score": 72.5,
  "signal": "STRONG_BUY",
  "factors": {
    "momentum": 68.2,
    "mean_reversion": 75.0,
    "breakout": 70.5,
    "volatility": 76.3
  }
}
```

**文件**: `v4_3/factor_score.py`

---

### 2.3 Risk Agent

**职责**: 独立风控，实时否决权

**输入**:
- 账户信息
- 当前持仓
- 交易信号（来自 Factor Score）

**处理**:
1. 账户级检查（保证金、杠杆）
2. 仓位检查（数量、品种）
3. 风险敞口检查（总风险≤3%）
4. VaR 检查（95% VaR ≤ 2% 净值）
5. 相关性检查

**输出**:
```json
{
  "can_trade": true,
  "decisions": [
    {"check": "account_level", "passed": true},
    {"check": "positions", "passed": true},
    {"check": "risk_exposure", "passed": true},
    {"check": "var", "passed": true},
    {"check": "correlation", "passed": true}
  ],
  "suggested_volume": 0.15
}
```

**文件**: `v4_3/risk_agent.py`

---

### 2.4 Execution Agent

**职责**: 订单执行

**输入**:
- 交易信号（通过风控）
- 建议仓位

**处理**:
1. 计算止损止盈（基于 ATR）
2. 下单（MT5 API）
3. 记录订单

**输出**:
```json
{
  "success": true,
  "order_id": 123456789,
  "entry_price": 1.23456,
  "sl": 1.23156,
  "tp": 1.23756
}
```

**文件**: `v4_3/main_v43.py`（执行部分）

---

### 2.5 Review Agent

**职责**: 自动复盘，归因分析

**输入**:
- 交易历史（数据库）
- 因子数据

**处理**:
1. 归因分析（策略/品种/时间/因子）
2. 问题识别
3. 改进建议

**输出**:
```markdown
# 每日复盘报告
- 交易次数：5 单
- 胜率：60%
- 总盈亏：+$125.50
- 问题分析：...
- 改进建议：...
```

**文件**: `v4_3/review_agent.py`

---

### 2.6 Walk-Forward Validator

**职责**: 滚动回测验证

**输入**:
- 历史 K 线数据
- 参数范围

**处理**:
1. 生成滚动窗口
2. 训练集优化参数
3. 测试集验证
4. 计算稳定性评分

**输出**:
```json
{
  "windows_count": 6,
  "avg_train_sharpe": 1.85,
  "avg_test_sharpe": 1.42,
  "stability_score": 85,
  "consistency_score": 78,
  "total_score": 81.5,
  "passed": true
}
```

**文件**: `v4_3/walk_forward.py`

---

### 2.7 Factor IC Analyzer

**职责**: 因子有效性检验

**输入**:
- 历史 K 线数据
- 因子值

**处理**:
1. 计算 IC（相关系数）
2. 显著性检验（t 检验）
3. 衰减分析
4. 权重优化

**输出**:
```json
{
  "factors": {
    "momentum": {"ic": 0.08, "p_value": 0.02, "rating": "有效"},
    "rsi": {"ic": 0.05, "p_value": 0.05, "rating": "有效"}
  },
  "optimal_weights": {
    "momentum": 0.62,
    "rsi": 0.38
  }
}
```

**文件**: `v4_3/factor_ic.py`

## 三、数据流

### 3.1 实时交易流程

```
[MT5 数据] 
    │
    ▼
[Market Regime] ────→ 市场状态 + 动态阈值
    │
    ▼
[Factor Score] ──────→ 因子评分 + 交易信号
    │
    ▼
[Risk Agent] ────────→ 风控检查 + 仓位建议
    │
    ▼
[Execution Agent] ───→ MT5 下单
    │
    ▼
[数据库] ────────────→ 交易记录
```

### 3.2 复盘流程

```
[数据库]
    │
    ▼
[Review Agent] ──────→ 归因分析 + 问题识别
    │
    ▼
[报告生成] ──────────→ Markdown 报告
```

### 3.3 验证流程

```
[历史数据]
    │
    ▼
[Walk-Forward] ──────→ 滚动回测
    │
    ▼
[Factor IC] ─────────→ 因子有效性
    │
    ▼
[验收决策] ──────────→ Go/No-Go
```

## 四、配置管理

### 4.1 配置文件

| 文件 | 用途 | 关键参数 |
|------|------|----------|
| `regime_config.json` | 市场状态判断 | ADX 阈值、ATR 倍数 |
| `factor_weights.json` | 因子权重 | 各因子权重、评分阈值 |
| `risk_params.json` | 风控参数 | 最大杠杆、风险上限 |
| `walk_forward_config.json` | 回测配置 | 训练/测试月数、参数范围 |

### 4.2 参数调优

**市场状态参数**:
```json
{
  "adx_threshold": 25,        // ↑更严格识别趋势
  "atr_multiplier": 1.5       // ↑更严格识别高波动
}
```

**因子权重**:
```json
{
  "momentum": 0.30,           // 根据 IC 调整
  "mean_reversion": 0.30      // 根据 IC 调整
}
```

**风控参数**:
```json
{
  "max_leverage": 3.0,        // ↓更保守
  "risk_per_trade": 0.005     // ↓更保守
}
```
