# 海龟交易策略 (Turtle Trading)

> 策略来源：聚宽 #49 | 学习时间：2026-03-18

---

## 📊 策略概述

**历史背景**: 1983 年 Richard Dennis 培训交易员的著名实验

**核心思路**: 趋势跟踪 + 突破入场 + ATR 仓位管理

**适用市场**: 股票、外汇、期货、加密货币

**时间框架**: D1/H4 (外汇)

---

## 🧮 海龟交易核心规则

### 1. 入场信号 (突破)

```
系统 1 (短期):
- 做多：价格突破过去 20 日高点
- 做空：价格跌破过去 20 日低点

系统 2 (长期):
- 做多：价格突破过去 55 日高点
- 做空：价格跌破过去 55 日低点
```

### 2. 加仓规则 (金字塔)

```
初始仓位：1 单位

每上涨 0.5×ATR: 加仓 1 单位
最多加仓：3 次 (总仓位 4 单位)

示例 (做多):
入场：1.2000
加仓 1: 1.2050 (+0.5×ATR)
加仓 2: 1.2100
加仓 3: 1.2150
```

### 3. 止损规则

```
统一止损：2×ATR

做多止损：入场价 - 2×ATR
做空止损：入场价 + 2×ATR

所有仓位统一止损
```

### 4. 止盈规则

```
系统 1 出场:
- 做多：价格跌破过去 10 日低点
- 做空：价格突破过去 10 日高点

系统 2 出场:
- 做多：价格跌破过去 20 日低点
- 做空：价格突破过去 20 日高点
```

### 5. 仓位计算

```
单位仓位 = (账户风险%) / (N 值 × 每点价值)

N 值 = ATR(14)

示例:
账户：$100,000
风险：1% = $1,000
ATR(N): 50 点
每点价值：$10

单位仓位 = 1000 / (50 × 10) = 2 手
```

## ⚠️ 注意事项

### 海龟策略的优点

1. **简单明确**: 规则清晰，易于执行
2. **趋势捕捉**: 大趋势时盈利丰厚
3. **风险控制**: ATR 仓位管理科学
4. **可复制**: 机械化，无需主观判断

### 海龟策略的缺点

1. **震荡市亏损**: 横盘时频繁止损
   - 解决：用 ADX 过滤，ADX<25 不交易

2. **胜率低**: 通常只有 35-45%
   - 心理挑战：连续止损后能否坚持

3. **回撤大**: 可能连续亏损 10 次以上
   - 解决：降低单笔风险 (1% → 0.5%)

4. **加仓风险**: 趋势反转时亏损放大
   - 解决：严格止损，不扛单

## 🔧 策略改进方向

### 改进 1: ADX 过滤

```
只在大趋势时交易:
ADX > 30 → 允许交易
ADX < 25 → 观望

优势：减少震荡市亏损
劣势：可能错过趋势初期
```

### 改进 2: AMA 替代突破

```
用 AMA 方向替代固定周期突破:
AMA 向上 + 价格突破 → 做多
AMA 向下 + 价格突破 → 做空

优势：自适应，减少滞后
```

### 改进 3: 部分止盈

```
2 单位仓位:
- 1 单位在 2R 止盈
- 1 单位追踪止损 (让利润奔跑)

优势：锁定部分利润，心态更好
```

## 📝 与策略 2.0 的融合

### 可能的结合点

1. **入场信号**: 用 AMA 替代固定周期突破
   ```
   AMA 向上 + 价格突破 N 日高点 → 做多
   ```

2. **仓位管理**: 沿用海龟的 ATR 仓位计算
   ```
   单位仓位 = 风险金额 / (ATR × 每点价值)
   ```

3. **加仓规则**: 保留金字塔加仓
   ```
   每 0.5×ATR 加仓一次
   ```

4. **追踪止损**: 结合吊灯止损
   ```
   初始止损：2×ATR
   盈利后：吊灯止损 (最高价 - 3×ATR)
   ```

## 📋 交易检查清单

### 开仓前

- [ ] ADX > 25 (趋势强度)
- [ ] 价格突破 N 日高点/低点
- [ ] AMA 方向确认
- [ ] 无重大经济事件

### 加仓条件

- [ ] 现有仓位盈利
- [ ] 价格上涨 0.5×ATR
- [ ] 未达到最大加仓次数 (4 次)
- [ ] 趋势仍在延续

- [ ] 触及止损 (2×ATR)
- [ ] 出场信号 (跌破 N 日低点)
- [ ] AMA 反向
