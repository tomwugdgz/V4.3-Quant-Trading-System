# RSRS 指标优化版 - 成交量加权

## 📊 策略概述

**RSRS **(Resistance Support Relative Strength)

**核心思路**: 
1. 基础 RSRS: 高低点线性回归斜率
2. 加入 R² (拟合优度) 加权
3. 加入成交量加权
4. 得到更准确的趋势信号

**适用市场**: 股票、指数、外汇、期货

**时间框架**: D1 (原策略), 可适配 H4

---

## 🧮 指标计算

### 基础 RSRS

```python
1. 获取 N 日最高价和最低价 (默认 N=18)
2. 用最低价对最高价做线性回归
   High = β × Low + α
3. β (斜率) = 基础 RSRS 值
```

### 优化版 RSRS (带权)

```python
1. 计算基础 RSRS (β)
2. 计算 R² (拟合优度，0-1)
3. 计算成交量比率 = 近期成交量 / 平均成交量
4. 加权 RSRS = β × R² × 成交量比率 × Z-Score

参数:
- N: 18 (回归窗口)
- M: 1100 (历史窗口，用于 Z-Score 计算)
- 成交量窗口：9 日
```

### 交易信号

```
买入信号：加权 RSRS > 0.7
卖出信号：加权 RSRS < -0.7

阈值可调整:
- 激进：±0.5
- 中性：±0.7 (默认)
- 保守：±1.0
```

## 💡 与基础 RSRS 的对比

| 特性 | 基础 RSRS | 优化版 RSRS |
|------|----------|------------|
| 计算 | 单纯斜率β | β × R² × 成交量 |
| 信号质量 | 一般 | 更好 (多重加权) |
| 假信号 | 较多 | 较少 |
| 滞后性 | 中等 | 略高 (计算复杂) |
| 适用场景 | 所有市场 | 更适合高波动市场 |

### 外汇成交量替代方案

```
外汇无统一成交量，可用替代指标:
1. Tick Volume (MT5 提供)
2. 波动率 (ATR)
3. 价格变化幅度

替代方案:
加权 RSRS = β × R² × (ATR / 平均 ATR)
```

## 🔧 策略改进方案

### 方案 1: 简化版 (无成交量)

```python
# 外汇简化版
加权 RSRS = β × R² × Z-Score

买入：加权 RSRS > 0.8
卖出：加权 RSRS < -0.8

优点：计算简单，适合外汇
缺点：缺少成交量确认
```

### 方案 2: 波动率加权

```python
ATR_ratio = ATR(14) / 平均 ATR(50)
加权 RSRS = β × R² × ATR_ratio × Z-Score

买入：加权 RSRS > 0.8
卖出：加权 RSRS < -0.8

优点：用波动率替代成交量
缺点：ATR 滞后
```

### 方案 3: 多时间框架确认

```
D1: 计算加权 RSRS 方向
- RSRS > 0 → 只做多
- RSRS < 0 → 只做空

H4: 生成交易信号
- 加权 RSRS 上穿 0.8 → 做多
- 加权 RSRS 下穿 -0.8 → 做空
```

### 方案 4: RSRS+AMA 组合

```
趋势确认：AMA 向上
RSRS 信号：加权 RSRS > 0.8
入场：两者同时满足

出场:
- 加权 RSRS < 0
- 或 吊灯止损

优点：双重确认，信号质量高
缺点：入场机会较少
```

## ⚠️ 注意事项

### 优化版 RSRS 的优点

1. **信号质量高**: 多重过滤，假信号少
2. **考虑拟合优度**: R² 低时降低权重
3. **考虑成交量**: 放量时信号更可靠

### 优化版 RSRS 的缺点

1. **计算复杂**: 需要线性回归 + 统计计算
2. **外汇适配难**: 成交量数据不统一
3. **参数敏感**: 不同市场需要调整

### 风险控制

```
单笔风险：≤ 2%
止损设置:
- 固定止损：2-3×ATR
- 技术止损：前低/前高

最大持仓：2-3 个货币对
```
