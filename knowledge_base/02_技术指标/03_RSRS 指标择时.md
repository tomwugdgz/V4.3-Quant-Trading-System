## 📊 策略概述

**RSRS **(Resistance Support Relative Strength) - 阻力支撑相对强度指标

**核心思路**: 通过线性回归分析高低点关系，预测趋势方向

**适用市场**: 股票、指数、外汇、期货

**时间框架**: 30 分钟 K 线 (原策略), 可适配 H4/D1

---

## 🧮 指标原理

### RSRS 计算方法

```python
1. 获取 N 日最高价和最低价序列 (默认 N=18)
2. 用最低价对最高价做线性回归: High = β × Low + α
3. β (斜率) 就是 RSRS 指标值
   - β > 0: 上涨趋势
   - β < 0: 下跌趋势
4. 对β进行标准化处理 (Z-Score)
   - 减去均值，除以标准差
   - 得到 beat_z 值
```

### 交易信号

```
买入信号：beat_z 从 < 0.7 上穿 > 0.7
卖出信号：beat_z 从 > -0.7 下穿 < -0.7

阈值可调整:
- 激进：±0.5
- 中性：±0.7 (默认)
- 保守：±1.0
```

## 🔧 策略改进方向

### 改进 1: 动态阈值

```python
# 根据波动率调整阈值
ATR_ratio = ATR(14) / Close
if ATR_ratio > 0.01:  # 高波动
    threshold = 0.9
else:  # 低波动
    threshold = 0.7
```

### 改进 2: 多时间框架确认

```
D1 RSRS > 0 → 只做多
H4 RSRS 上穿阈值 → 入场
H1 回调 → 精确入场点
```

### 改进 3: 结合其他指标

```
RSRS + ADX:
- RSRS 上穿 0.8
- ADX > 25 (趋势强度)
- 两者同时满足才入场

RSRS + AMA:
- RSRS 确认方向
- AMA 确认趋势
- 双重过滤
```

### 改进 4: 止损优化

```
固定止损：2-3×ATR
技术止损：前低/前高
追踪止损：吊灯止损 (3×ATR)

止盈:
- 第一目标：2R
- 第二目标：RSRS 反向信号
```

## ⚠️ 注意事项

### RSRS 的优点

1. **前瞻性**: 基于高低点关系，有一定预测性
2. **客观**: 纯数学计算，无主观判断
3. **适应性强**: 可用于不同市场和时间框架

### RSRS 的缺点

1. **滞后性**: 基于历史数据，极端行情滞后
2. **震荡市失效**: 无明显趋势时假信号多
3. **参数敏感**: 不同市场需要调整参数

### 风险控制

```
单笔风险：≤ 2%
每日止损：≤ 5%
最大持仓：2-3 个货币对
相关性控制：避免同时持有高相关货币对
```
