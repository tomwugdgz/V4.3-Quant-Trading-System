## 📊 策略概述

**均值回归策略** - 价格偏离均线过大时反向交易

**核心思路**: 
1. 计算价格与移动平均的差值
2. 对差值做 Z-Score 标准化
3. Z-Score 过低 (超卖) → 买入
4. Z-Score 过高 (超买) → 卖出

**适用市场**: 股票、外汇、期货

**时间框架**: D1 (原策略), 可适配 H4

---

## 🧮 策略逻辑

### Z-Score 计算

```python
1. 计算移动平均 (MA, 默认 20 日)
2. 计算价格与均线的差值：sub = Close - MA
3. 计算 sub 的均值和标准差
4. Z-Score = (sub - mean) / std

参数:
- ma_window: 20 (均线周期)
- zscore_window: 60 (Z-Score 计算窗口)
```

### 交易信号

```
买入条件:
- Z-Score ≤ -2 (价格低于均线 2 个标准差)
- 当前无持仓

卖出条件:
- Z-Score ≥ 1 (价格回归均线上方)
- 当前有持仓

参数:
- lower: -2 (买入阈值)
- upper: 1 (卖出阈值)
```

### 仓位管理

```
最大持仓：1 只股票
仓位分配：可用资金 / (最大持仓 - 已持仓)
```

## 🔧 策略改进方案

### 方案 1: 动态阈值

```python
# 根据波动率调整阈值
ATR_ratio = ATR(14) / Close
if ATR_ratio > 0.01:  # 高波动
    lower = -3.0
    upper = 2.0
else:  # 低波动
    lower = -2.0
    upper = 1.0
```

### 方案 2: 多时间框架确认

```
D1: 确定主趋势
- Z-Score > 0 → 只做空 (价格偏高，等待回调)
- Z-Score < 0 → 只做多 (价格偏低，等待反弹)

H4: 生成交易信号
- Z-Score < -2.5 → 入场做多
- Z-Score > 1.5 → 入场做空
```

### 方案 3: Z-Score+AMA 组合

```
趋势确认：AMA 走平 (震荡市)
入场信号：Z-Score < -2.5 (超卖)
出场信号：Z-Score > 0 (回归均值)

优点：只在震荡市使用，避开趋势市
缺点：需要判断市场状态
```

### 方案 4: 分批建仓

```
第一批：Z-Score < -2.0, 仓位 30%
第二批：Z-Score < -2.5, 仓位 30%
第三批：Z-Score < -3.0, 仓位 40%

出场:
- 第一批：Z-Score > 0
- 第二批：Z-Score > 0.5
- 第三批：Z-Score > 1.0

优点：降低平均成本
缺点：管理复杂
```

## ⚠️ 注意事项

### 均值回归的优点

1. **逻辑清晰**: 价格总会回归均值
2. **震荡市表现好**: 高抛低吸
3. **参数简单**: 易于理解和实现

### 均值回归的缺点

1. **趋势市亏损**: 强趋势时价格可能不回归
2. **需要过滤**: 必须识别震荡市 vs 趋势市
3. **止损重要**: 防止价格继续偏离

### 风险控制

```
单笔风险：≤ 2%
止损设置:
- 固定止损：2-3×ATR
- 技术止损：前低/前高
- 时间止损：N 根 K 线不回归则出场

最大持仓：2-3 个货币对
相关性控制：避免高相关货币对
```
